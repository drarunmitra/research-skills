Nine composable skills that carry a study from raw data to a citable, published paper.

Built for public-health and medical research, and written to run the same on Claude, ChatGPT, and Gemini. Analysis, academic writing, peer review, citations, reproducibility, teaching, and the occasional blog post. Each skill knows where it sits and which sibling to hand off to.

02

The catalogue

Nine skills in five groups. Each entry lists what it does and a phrase that triggers it.
Analysis
01
time-series
Analyse and forecast time series in R with the tidyverts stack (tsibble, feasts, fable): decomposition, ETS/ARIMA/dynamic regression, distributional forecasts, cross-validation, plus surveillance and outbreak-detection layers.
Triggers on "forecast these case counts", "ETS or ARIMA?", "detect an outbreak"
Environment
02
qual-researcher
Qualitative analysis end to end: transcript cleaning, coding, reflexive thematic analysis (Braun & Clarke), framework mapping and synthesis, with COREQ/SRQR reporting and every theme traced to a verbatim quote.
Triggers on "code this transcript", "thematic analysis", "map to COM-B"
Any LLM
Writing & review
03
research-writer
Draft, structure and revise academic writing of any kind: theses, journal manuscripts, grant proposals and reports. IMRaD templates, argument discipline, and 23 named writing-problem fixes.
Triggers on "draft the introduction", "structure my paper", "review this chapter"
Any LLM
04
academic-writer
The canonical humaniser: strip AI tells, calibrate every claim to the strength of its evidence, verify or flag citations rather than fabricate them, and convert citation styles. A revision skill, run last.
Triggers on "humanise this", "remove the AI bloat", "check my citations"
Any LLM
05
peer-reviewer
Read a manuscript as a journal editor and reviewer would: an editorial recommendation, prioritised comments tied to the text, a methods-and-statistics critique against the right reporting guideline, and a claim-versus-evidence audit.
Triggers on "what would a reviewer say", "will this get rejected", "pre-submission review"
Any LLM
Citations & reproducibility
06
zotero-cite
Find and verify a reference, add it to Zotero through the local API, export the collection to a .bib, and cite it from Quarto, R Markdown or LaTeX with @citekey. Keeps the bibliography reproducible.
Triggers on "add this paper to Zotero and cite it", "keep a .bib in sync"
Environment
07
reproducible-repo
Package a finished analysis (code, data and manuscript) into a public, citable, reproducible repository and archive it for a DOI. Handles non-redistributable data, large binaries, CITATION.cff, .zenodo.json and Quarto front-matter.
Triggers on "make this reproducible", "get a Zenodo DOI", "prepare for medRxiv"
Environment
Teaching
08
slides-creator-skill
Build and publish teaching decks as Quarto Reveal.js presentations in a deliberately minimal house style: stock defaults plus a tiny stylesheet, with structure from native Quarto constructs, never bespoke CSS.
Triggers on "make slides for", "de-fancify this deck", "publish to GitHub Pages"
Environment
Non-academic content
09
content-research-writer
A writing partner for non-academic content: blogs, newsletters and thought leadership. Collaborative outlining, sharper hooks, section-by-section feedback, and your voice preserved. Fenced off from the academic track.
Triggers on "write a blog post", "improve this hook", "outline an article"
Any LLM

Install & use

Each skill is one SKILL.md. The frontmatter is for Claude; ChatGPT and Gemini just read the body.

Clone the repo, then symlink (or copy) the skill folders into ~/.claude/skills/. Claude Code auto-discovers them and triggers each at the right moment.

# clone
git clone https://github.com/drarunmitra/research-skills.git
# symlink one skill into your skills dir
ln -s "$PWD/research-skills/research-writer" ~/.claude/skills/research-writer

On Windows, enable Developer Mode for symlinks, or copy the folder instead.

Open a Project, then paste the body of a skill's SKILL.md into the Project instructions. The YAML frontmatter at the top can stay or be trimmed; claude.ai reads the prose.

Best for the five pure-reasoning skills. The four environment skills need a local shell, so use Claude Code for those.

Create a Custom GPT (or a Project) and paste the skill's SKILL.md body into its instructions. The skill behaves the same; only the tool names differ.

For the environment-dependent skills, use Codex / agent mode with a connected environment so the shell steps can run.

Create a Gem from the skill's SKILL.md body, or place it under the Gemini CLI's skill mechanism.

Pure-reasoning skills work in the Gemini app; the tooled skills need the Gemini CLI for their shell steps.
